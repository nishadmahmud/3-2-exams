**1.** What is cryptography? Briefly explain **CIA triad** of computer Security. **(5)**
> **Cryptography:** The science and art of securing communication and data by transforming it into an unintelligible format (ciphertext) so that only authorized parties can read it using a specific key.
> 
> **CIA Triad:**
> *   **Confidentiality:** Ensuring that information is only accessible to authorized individuals (e.g., via encryption).
> *   **Integrity:** Ensuring that information remains accurate and unaltered from its original state by unauthorized individuals (e.g., via hashing and MACs).
> *   **Availability:** Ensuring that information and systems are available and accessible to authorized users when needed.

**2.** Using the **Extended Euclidean algorithm**, determine the **GCD of 305 and 85** as well as the value of **x and y**. **(5)**
> The equation is $\gcd(305, 85) = 305x + 85y$.
> 
> **Step 1: Find GCD**
> *   $305 = 3 \times 85 + 50$
> *   $85 = 1 \times 50 + 35$
> *   $50 = 1 \times 35 + 15$
> *   $35 = 2 \times 15 + 5$
> *   $15 = 3 \times 5 + 0$.
> Thus, the **$\gcd(305, 85) = 5$**.
> 
> **Step 2: Back substitution to find x and y**
> *   $5 = 35 - 2(15)$
> *   $5 = 35 - 2(50 - 1(35)) = 3(35) - 2(50)$
> *   $5 = 3(85 - 1(50)) - 2(50) = 3(85) - 5(50)$
> *   $5 = 3(85) - 5(305 - 3(85)) = 18(85) - 5(305)$
> 
> Rearranging into the form $305x + 85y$:
> $5 = 305(-5) + 85(18)$
> Therefore, **$x = -5$** and **$y = 18$**.

**3.** Using **Fermat's Theorem**, examine whether the number **43** is a prime number or not. **(5)**
> **Fermat's Little Theorem:** If $p$ is prime and $\gcd(a, p) = 1$, then $a^{p-1} \equiv 1 \pmod p$.
> 
> Let $p = 43$. We will test with $a = 2$. We need to compute $2^{42} \pmod{43}$ using repeated squaring:
> *   $2^1 \equiv 2 \pmod{43}$
> *   $2^2 \equiv 4 \pmod{43}$
> *   $2^4 \equiv 16 \pmod{43}$
> *   $2^8 = 256 \equiv -2 \pmod{43}$ *(Since $43 \times 6 = 258$, $256 = 258 - 2$)*
> *   $2^{16} \equiv (-2)^2 = 4 \pmod{43}$
> *   $2^{32} \equiv 4^2 = 16 \pmod{43}$
> 
> Since $42 = 32 + 8 + 2$:
> $2^{42} \equiv 2^{32} \times 2^8 \times 2^2 \pmod{43}$
> $2^{42} \equiv 16 \times (-2) \times 4 \pmod{43}$
> $2^{42} \equiv -128 \pmod{43}$
> Since $43 \times 3 = 129$, $-128 \equiv -128 + 129 = \mathbf{1 \pmod{43}}$.
> 
> Since $2^{42} \equiv 1 \pmod{43}$, the test passes, and we conclude that 43 is **probably a prime number**.

**1.** Given, two prime number **p and q are 7 and 19** respectively, generate pubic key and private key pair using RSA algorithm where **e = 17**. **(6)**
> **Step 1: Find n and $\varphi(n)$**
> *   $n = p \times q = 7 \times 19 = \mathbf{133}$
> *   $\varphi(n) = (p-1)(q-1) = 6 \times 18 = \mathbf{108}$
> 
> **Step 2: Define Public Key**
> *   The public key is $(e, n) = \mathbf{(17, 133)}$.
> 
> **Step 3: Calculate Private Key (d)**
> *   We need $17 \times d \equiv 1 \pmod{108}$. Use Extended Euclidean:
> *   $108 = 6(17) + 6$
> *   $17 = 2(6) + 5$
> *   $6 = 1(5) + 1$
> *   Substitute back:
>     $1 = 6 - 5$
>     $= 6 - (17 - 2(6)) = 3(6) - 17$
>     $= 3(108 - 6(17)) - 17 = 3(108) - 19(17)$
> *   So, $d = -19 \equiv 108 - 19 = \mathbf{89}$.
> *   The private key is $(d, n) = \mathbf{(89, 133)}$.

**2.** What are the required characteristics of public key cryptography? **(4)**
> *   It is computationally easy for a party to generate a key pair (public key, private key).
> *   It is computationally easy for a sender to generate ciphertext using the receiver's public key.
> *   It is computationally easy for the receiver to decrypt the ciphertext using their own private key.
> *   It is computationally infeasible for an adversary to determine the private key from the public key.
> *   It is computationally infeasible for an adversary to recover the original message from the public key and ciphertext alone.
> *   *(Optional but useful for signatures)*: Either of the two related keys can be used for encryption, with the other used for decryption.

**3.** Define **collision resistant** and **cryptographic checksum**. **(3)**
> **Collision Resistant:** A property of a cryptographic hash function making it computationally infeasible to find any two distinct, different messages $x$ and $y$ that produce the exact same hash output ($H(x) = H(y)$).
> 
> **Cryptographic Checksum:** Also known as a Message Authentication Code (MAC), it is a fixed-size block of data generated from a message using a shared secret key. It is attached to the message to ensure data integrity and verify the origin's authenticity.

**4.** How to get data confidentiality along with authentication and integrity using **Digital Signature**? **(7)**
> To achieve all three properties, digital signatures must be combined with public-key encryption:
> 1.  **Authentication & Integrity (Signing):** The sender first creates a hash of the plaintext message and encrypts the hash using their *own private key*. This signature proves the sender's identity and ensures the message hasn't been altered.
> 2.  **Confidentiality (Encryption):** The sender then encrypts the original message and the attached signature together using the receiver's *public key*. 
> 
> The receiver decrypts the entire package with their private key (ensuring confidentiality), and then verifies the signature with the sender's public key (ensuring authentication and integrity).
